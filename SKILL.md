# Benchmark Analyzer — 对标账号全自动拆解

## 你的任务

用户给你一个**【社媒助手】导出的 Excel 数据文件**，你必须**不跳过任何步骤**地完成从数据到分析的全流程，最终输出标准化的对标账号案例库文件夹。

---

## 执行流程（严格执行，不允许跳过任何一步）

### 步骤 1：读取Excel并运行自动化脚本

1. 先确认用户提供的 Excel 文件路径
2. 运行 `scripts/benchmark_analyzer.py` 脚本。**注意：因为 skill 的安装目录和用户当前工作目录可能不同，必须使用绝对路径调用**：
   ```bash
   python3 ~/.claude/skills/benchmark-analyzer/scripts/benchmark_analyzer.py "<Excel文件路径>"
   ```
   如果当前项目已经有 `对标账号案例库/` 文件夹，脚本会自动将结果输出到该目录下；否则会在当前工作目录新建 `对标账号案例库/`。
3. 脚本运行完成后，会生成 `对标账号案例库/<博主名>/` 文件夹，内含：
   - `原始数据/` — Excel备份
   - `爆款内容体检报告_<博主名>.md`
   - `音频/` — Top10音频MP3
   - `语料/` — 原始转写语料（.md格式）

### 步骤 2：AI纠错（静默执行）

读取 `语料/` 文件夹下的所有 `.md` 文件，执行以下操作**直接覆盖原文件**，不需要向用户汇报改了哪些字：

1. **繁简转换**：将台湾/香港繁体字转为简体字
2. **错别字修正**：根据上下文判断并修正whisper转写错误（如：品位→品类、负构好→复购好、来钱卖→来钱慢、这把对话→这八句话等）
3. **格式统一**：确保每个语料文件保持 `标题：xxx
文案：
---
[正文]` 的格式
4. **细节清理**：去除多余空格、修正明显的断句错误

### 步骤 3：生成文案分析报告

基于修正后的 Top10 语料，生成 `文案分析报告.md`，必须包含以下 10 个章节：

1. **开头钩子分析** — 逐条拆解前3秒抓人技巧，归纳规律
2. **标题公式拆解** — 拆解标题结构、标签组合规律
3. **内容结构模式** — 归纳2-3套可复用的内容模板
4. **情绪触发器** — 焦虑/欲望/身份认同触发方式
5. **金句提取** — 分类整理对立型/方法论型/情感型金句
6. **结尾设计** — 总结升华型/行动号召型结尾规律
7. **可复用文案框架** — 给出可直接套用的文案公式
8. **关键话术词汇表** — 按类别整理高频词
9. **执行层面参考** — 给出5条以上可直接落地的操作建议
10. **AI复刻指南** — 包含：必喂信息清单、提示词模板、关键指标

### 步骤 4：与现有技能协同（如用户需要）

如果当前项目是 **content-os** 内容生产系统（存在 `IP_Profiles.md`、`agents/` 等），主动询问用户是否需要把对标结果用于下一步：
- **topic-miner**：基于爆款主题分布和高频词，生成新的选题建议
- **content-researcher**：基于对标账号的高互动话题方向，搜集相关素材
- **auto-pilot**：用对标账号的语料+分析报告作为参考，直接生成一篇同风格文章

在 `文案分析报告.md` 末尾追加一个章节：

```markdown
## 十一、下游使用建议

### 可用于 topic-miner 的选题方向
- {从高频主题中提炼的 3-5 个选题方向}

### 可用于 content-researcher 的搜索关键词
- {从爆款标题和标签中提取的搜索词}

### 可用于 auto-pilot 的语料包
- {Top10 语料文件夹路径}
- {爆款内容体检报告文件路径}
```

### 步骤 5：完成汇报

向用户汇报：
1. 案例库文件夹的完整路径
2. 生成了哪些文件
3. 该账号的核心特征（3-5条关键洞察）
4. 建议的下一步动作（如：继续用 topic-miner 找选题 / 用 auto-pilot 写稿 / 用 content-researcher 搜素材）

## 输出格式规范

### 语料文件格式

```markdown
标题：<视频标题>
文案：
---
[修正后的纯净口语化文案]
```

### 文案分析报告的章节标题

使用 `#` 到 `###` 的层级结构，多用表格和列表，少用长段落。

---

## 注意事项

- **绝不跳过 AI 纠错步骤**。转写语料必须 readable 才能生成有价值的分析。
- **纠错时不需要逐条汇报**。默默地读、默默地改、默默地覆盖保存。
- **如果音频下载失败**，在报告中注明"X条音频缺失"，不影响其余流程。
- **如果脚本运行报错**，优先尝试修复环境问题（如模型缺失、路径错误），再重新执行。
- **路径兼容**：调用脚本时必须使用 `~/.claude/skills/benchmark-analyzer/scripts/benchmark_analyzer.py` 的绝对路径，不能用相对路径 `scripts/...`。
